Bus de données

Le bus de données transporte la valeur qui circule dans le MCU: une instruction, une variable en RAM, un octet reçu en UART, ou un registre de périphérique. Si le bus d’adresse dit où aller, le bus de données dit quoi transférer.

Transfert

Rôle dans le système: pourquoi et comment

Une fois qu’une adresse est choisie (via le bus d’adresse), il faut bien déplacer la donnée. C’est le rôle du bus de données: il sert de “route” pour transporter les bits entre le CPU, la mémoire (RAM/Flash) et les périphériques (I/O).

Il fonctionne dans les deux sens: en lecture, la mémoire ou le périphérique envoie la valeur vers le CPU. En écriture, le CPU envoie une valeur vers la mémoire ou un registre de périphérique. Les signaux de contrôle (lecture/écriture) indiquent qui parle et quand.

Question
“Quoi ?” (la valeur)
Direction
Lecture (vers CPU) / Écriture (depuis CPU)
Avec
Bus d’adresse + contrôle (R/W)

Exemple concret

Le CPU veut écrire la valeur 0x3A en RAM à l’adresse 0x2000. L’adresse est placée sur le bus d’adresse, puis 0x3A est placée sur le bus de données, et enfin le signal “écriture” valide l’opération.

Adresse = 0x2000 → Écriture → Donnée = 0x3A

Plus de détails

La largeur du bus de données (8, 16, 32 bits…) détermine combien de bits peuvent être transférés en une seule fois. Par exemple, un bus 8 bits transfère un octet à la fois, tandis qu’un bus 32 bits peut transférer 4 octets en une seule opération (selon l’architecture).

Dans beaucoup de microcontrôleurs, les périphériques sont en memory-mapped I/O: lire/écrire un registre UART, GPIO ou timer ressemble à lire/écrire une case mémoire. Ça simplifie le modèle: adresse → transfert de données.

Attention à une confusion fréquente: le bus de données ne “choisit” rien. Il transporte seulement la valeur. La sélection (la destination) est faite par le bus d’adresse, et le contrôle décide si on lit ou si on écrit.

Sans bus de données, le CPU pourrait pointer une adresse, mais il ne pourrait jamais recevoir une instruction ni stocker une variable. Autrement dit: pas de données, pas de calcul, pas de système.

TL;DR (à retenir)

Le bus de données transporte la valeur qui circule entre CPU, mémoire et périphériques. Il fonctionne en lecture et en écriture. L’adresse indique où, les données indiquent quoi, et le contrôle indique quand.

  • Données = “quoi”: la valeur transférée.
  • Bidirectionnel: lecture vers CPU, écriture depuis CPU.
  • Travaille avec: bus d’adresse + signaux R/W.
  • Largeur: influence la quantité transférée par opération.
