I/O (Entrées / Sorties)

Les I/O sont la partie du MCU qui communique avec l’extérieur: lire un bouton, mesurer un capteur, envoyer un signal PWM, piloter une LED, parler en UART/I2C/SPI. C’est le pont entre le code et le monde réel.

Interface matériel

Rôle dans le système: pourquoi et comment

Le CPU calcule, mais sans I/O il ne peut ni recevoir d’informations, ni agir. Les I/O offrent des broches GPIO (entrée/sortie) et des modules spécialisés: ADC (lecture analogique), PWM (contrôle de puissance), UART/I2C/SPI (communication), etc.

En pratique, le CPU configure les périphériques via des registres. Ensuite, il lit des valeurs (ex: état d’un bouton, résultat ADC) et écrit des commandes (ex: LED ON, signal PWM). Les interruptions permettent aussi de réagir vite (ex: un octet reçu en UART).

Entrées
Boutons, capteurs, signaux, données reçues
Sorties
LED, moteurs, relais, communication, PWM
Accès
Registres + buffers + interruptions

Exemple concret

Un bouton (entrée) déclenche l’allumage d’une LED (sortie). Le CPU lit une broche en entrée, puis écrit sur la broche de sortie.

if GPIO_IN == 1 → GPIO_OUT = 1
else → GPIO_OUT = 0

Plus de détails

Les entrées peuvent être instables sans résistances de tirage (pull-up/pull-down). Un bouton peut “rebondir” et provoquer plusieurs transitions rapides: on fait alors un anti-rebond (debounce) logiciel ou matériel.

Les sorties ont des limites: courant maximum par broche, tension logique, et puissance. Pour un moteur ou un relais, on utilise souvent un transistor ou un driver, car la broche du MCU ne fournit pas assez de courant. Le MCU fournit alors un signal de commande (ex: PWM) et le driver gère la puissance.

Les communications série utilisent des règles strictes de timing et souvent des buffers en RAM. Exemple: en UART, chaque octet reçu peut déclencher une interruption, stocker l’octet dans un buffer, puis le programme principal traite le message complet.

Conclusion: les I/O sont l’endroit où les erreurs de câblage et de configuration apparaissent le plus. Mais c’est aussi le bloc le plus “concret”, car c’est là que ton MCU interagit avec le vrai monde.

TL;DR (à retenir)

Les I/O permettent au MCU de lire (entrées) et agir (sorties). Le CPU configure des périphériques (GPIO, ADC, PWM, UART/I2C/SPI) via registres et échange des données. Les interruptions améliorent la réactivité en temps réel.

  • GPIO: entrée/sortie simple.
  • ADC: capteurs analogiques.
  • PWM: contrôle puissance/vitesse.
  • Comm: UART/I2C/SPI + buffers/interruptions.
