Bus d’adresse

Le bus d’adresse indique où le CPU veut accéder: une case mémoire (RAM/ROM) ou un registre de périphérique (I/O). C’est la “destination” d’une lecture ou d’une écriture.

Sélection

Rôle dans le système: pourquoi et comment

Dans un MCU, il existe de nombreuses zones: instructions en Flash, données en RAM, registres périphériques, etc. Pour accéder à la bonne zone, le CPU doit la sélectionner précisément. C’est exactement ce que permet le bus d’adresse.

Une opération complète implique généralement: adresse posée sur le bus d’adresse, type d’action (lecture/écriture), puis transfert de la valeur via le bus de données. Dans ton schéma, tu représentes l’idée de base, ce qui est parfait pour comprendre.

Question
“Où ?” (destination)
Cible
RAM / ROM(Flash) / registres I/O
Avec
Bus de données + contrôle

Exemple concret

Le CPU veut lire une variable en RAM. Il place l’adresse 0x2000 sur le bus d’adresse, puis lit la valeur correspondante sur le bus de données.

Adresse = 0x2000 → Lecture → Donnée = 0x3A

Plus de détails

Beaucoup de MCU utilisent le principe de memory-mapped I/O: les périphériques ont des adresses comme la mémoire. Exemple: un registre pour lire un port GPIO, un registre pour écrire une valeur PWM, un registre pour configurer l’UART, etc.

Grâce à ça, le CPU utilise toujours le même mécanisme: choisir une adresse puis lire/écrire des données. C’est simple et puissant, car le code peut interagir avec la mémoire et les périphériques de manière similaire.

La largeur du bus d’adresse (nombre de bits) influence le nombre total d’adresses possibles. Plus il y a de bits, plus on peut adresser de mémoire/périphériques. Pour ton niveau, retiens surtout l’idée: adresse = sélection.

Sans bus d’adresse, le CPU ne pourrait pas dire “je veux ce registre” ou “je veux cette variable”. Le MCU deviendrait un système incapable d’organiser ses ressources.

TL;DR (à retenir)

Le bus d’adresse sert à sélectionner quel endroit est ciblé (mémoire ou périphérique). Il ne transporte pas la valeur, seulement la destination. Une fois l’adresse choisie, les données sont transférées via le bus de données.

  • Adresse = “où”: destination.
  • Travaille avec bus de données + signaux de contrôle.
  • Permet d’accéder à RAM/Flash/I/O.
  • Concept clé: sélectionner puis transférer.
